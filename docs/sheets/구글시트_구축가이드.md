# 구글시트 구축 가이드 (6개 탭)

**✅ 시트 파일은 이미 회원님 구글 드라이브에 생성해 두었습니다.**
👉 **[평촌 한가람 A-5 동의율 모니터링 (열기)](https://docs.google.com/spreadsheets/d/1WCXj-jJaekpg-JQ4d8A3gLiNQ73EiImwXhIeqm4dRLc/edit)**

`Code.gs`의 **`setup`** 함수를 실행하면 아래 6개 탭이 자동 구성됩니다.

| 탭 | 역할 |
|----|------|
| **현황파악** | 동의율 + 의향(통합/제자리·신탁/조합)을 **수식으로 실시간 집계**(전체/단지별/동별) |
| **한양** | 아파트 동·호가 한 줄씩. 동의 **체크박스** + 통합/신탁 **드롭다운** + **필터** + 요약 |
| **삼성** | (동일) |
| **두산** | (동일) |
| **상가** | 단지별 상가 한 줄. **A열 = 단지 이름(한양/삼성/두산 드롭다운)**, 나머지 동일 |
| **단지정보** | 기본정보(구역명/마감/목표) + 공부상 세대수 + 동별 층수·라인수 |

> 💡 핵심 원칙: **분모(세대수)는 단지정보의 고정값**(한양 952·삼성 708·두산 436, 동별은 층수×라인수).
> 대표단은 **동의한 호를 기록**(동/호 + 동의 체크 + 통합/신탁 선택)하면 **동의수·동의율·의향%만** 변한다.
> 이름·연락처 같은 개인정보는 **입력하지 않는다**(다루는 건 동·호·숫자뿐).

---

## STEP 1. 코드 붙여넣기
1. 위 시트 열기 → 상단 **확장 프로그램 > Apps Script**
2. 기본 내용 지우고 이 폴더의 **`Code.gs` 전체 붙여넣기** → 저장(💾)

## STEP 2. `setup` 실행 → 6개 탭 자동 생성
1. 함수 드롭다운에서 **`setup`** 선택 → **실행 ▶** → 권한 승인
2. 시트로 돌아오면 **현황파악·한양·삼성·두산·상가·단지정보** 6탭 완성 ✅
   - 각 시트엔 헤더·체크박스·드롭다운·필터·요약 수식이 들어있음
   - 현황파악엔 단지별 동의 현황·의향 집계·동별 상세 수식이 들어있음
     (세대수 분모는 고정값이라 호가 없어도 952/708/436으로 표시됨)

> ⚠️ **예전 버전으로 이미 setup을 돌렸다면**: 코드 교체·저장 후 **`재설정` → `동의칸수정`**
> 순서로 실행하세요(아래 "업그레이드" 섹션 참고). 기존 데이터는 보존됩니다.

## STEP 3. (선택) 동별 층수·라인수 입력 → 동별 % 활성화
동별 동의율까지 보고 싶을 때만. `단지정보` 탭 아래쪽 **[■ 동별 구성]** 표에
각 동의 **층수**·**라인수(호/층)** 를 입력하면 그 동의 **세대수(분모)가 층수×라인수로 고정**된다.

| 단지 | 동 | 층수 | 라인수(호/층) | 비고 |
|------|----|-----|--------------|------|
| 한양 | 301 | 15 | 4 | ← 예시: 15층 × 4라인 = 60세대 |
| 한양 | 302 | ... | ... | |

- 미입력 동은 동별 동의율이 공란으로 표시(동의수는 계속 집계됨).
- 층수·라인수는 관리사무소(한양 031-386-0193 / 두산 031-382-3282)·현장 확인.
- 각 단지 (층수×라인수) 합이 공부상 세대수(한양 952·삼성 708·두산 436)와 맞는지 점검.

## STEP 4. (선택) `호생성` — 전체 호를 미리 깔기
동의한 호만 그때그때 기록하는 대신, **모든 호를 미리 깔아두고 체크**하는 방식을 원할 때만.
1. STEP 3을 채운 뒤 함수 **`호생성`** 실행 → 한양/삼성/두산에 동·호가 한 줄씩 생성(`호 = 층×100+라인`)
2. ⚠️ `호생성`은 해당 시트를 **다시 채우므로**(체크 초기화), 동의 입력을 시작한 뒤엔 재실행 금지.

## STEP 5. Web App 배포 (대시보드용 JSON API)
1. Apps Script 우측 상단 **배포 > 새 배포**
2. 유형(톱니바퀴) > **웹 앱** / 실행 계정: **나** / 액세스: **모든 사용자**
3. **배포** → 승인 → **웹 앱 URL 복사** (`https://script.google.com/macros/s/.../exec`)
4. 브라우저로 그 URL 접속 → JSON 이 보이면 성공. 이 URL을 대시보드에 연결.

## 운영 방법 (주민대표단)
- **동의서 접수 시**: 해당 단지 시트에서 그 호 행에 3가지 입력:
  1. **`동의` 체크박스 클릭**(✔) — `1`·`O` 등 글자 금지(체크박스 TRUE만 집계)
  2. **`통합/제자리`** 드롭다운에서 택1
  3. **`신탁/조합`** 드롭다운에서 택1
  - 호 찾기: 헤더의 **필터 버튼**(▽) → `호`/`동` 열에서 검색·선택. 또는 `Ctrl+F`.
  - (선택) `동의일자`·`비고` 기입.
- **현황파악**에서 동의율 + 통합/제자리·신탁/조합 **% 가 실시간 갱신**.
- **즉시 반영**: 체크하는 순간 `현황파악` 탭의 전체·단지·동별 수치가 자동 갱신.
  대시보드(웹)는 5~10분 캐시 후 갱신.
- **권한**: 편집 권한은 **주민대표단 계정만**. 시트 공유를 '뷰어'로 최소화.

## 이미 예전 버전으로 setup을 돌렸다면 (열/구조 업그레이드)
최신 `Code.gs` 붙여넣고 저장한 뒤 순서대로:
1. **`재설정`** → 현황파악을 최신 구조(의향 집계 포함)로 재생성 + 상가 시트 보강
2. **`동의칸수정`** → 한양/삼성/두산/상가 시트의 **열(통합·신탁 추가)·드롭다운·체크박스·필터·요약을
   데이터 보존한 채 최신화**
> ⚠️ 예전에 D열(동의일자)·E열(비고)에 값을 넣었다면, 새 구조에선 D=통합/제자리·E=신탁/조합이라
> 위치가 밀린다. 기존 입력이 거의 없다면 그대로 진행해도 무방(빈 칸이면 영향 없음).

## 각 탭 열 구조
**한양/삼성/두산 (아파트)**
| A | B | C | D | E | F | G |
|---|---|---|---|---|---|---|
| 동 | 호 | 동의(체크박스) | 통합/제자리(택1) | 신탁/조합(택1) | 동의일자 | 비고 |

**상가**
| A | B | C | D | E | F | G |
|---|---|---|---|---|---|---|
| **단지**(드롭다운) | 호 | 동의(체크박스) | 통합/제자리 | 신탁/조합 | 동의일자 | 비고 |
> 상가는 동 대신 **단지 이름**(한양/삼성/두산)을 A열에 넣는다.

> 세대주 요구 3항목: **동의(체크박스)** · **통합/제자리(드롭다운 택1)** · **신탁/조합(드롭다운 택1)**.
> C열 체크박스 · D/E열 드롭다운은 전체에 깔려 있음(행 추가해도 자동). 헤더 **필터**로 검색.
> 우측 H:I = 총세대수 / 동의수 / 동의율 요약(분모 고정).

## 현황파악 구성 (전부 수식)
1. **단지별 동의 현황** — 전체/한양/삼성/두산 세대수(고정)·동의수·동의율
2. **의향 집계** — 단지별 **통합/제자리**, **신탁/조합** 건수와 **%**(응답 세대 기준)
3. **동별 상세** — 동별 세대수·동의수·동의율(층수·라인수 입력 시 %표시)

## 단지정보 구성
① 기본설정(구역명/마감/목표/상가포함) ② 단지별 공부상 세대수 ③ 동별 구성(층수·라인수)

## 주의
- 코드 수정 후엔 **배포 > 배포 관리 > 편집(✏️) > 새 버전 > 배포**로 재배포해야 반영.
- 개인정보(이름/연락처/주소)는 **어떤 탭에도 입력하지 않는다.**
